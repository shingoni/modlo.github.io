Logging

Posted by Omar Baz on March 24, 2023

Logging refers to the mechanism of recording and storing information about system activities and events that occur within Moodle. It captures data such as user actions, system processes, error messages, and other relevant details. Enabling logging in your environment allows you to capture specific data that provides deeper insights into the functionality if your developed software during runtime. By analyzing the information, you can gain a better understanding of system behaviour, identify and resolve issues more effectively, and ensure the overall reliability and perfomance of your plugin.

To enable logging in Moodle, you first need to define what needs to be recorded; this could be anything from recording user logins to course enrolments. The following steps are:

  • Log into your Moodle site as an adminstrator
  • In Site Adminstration, navigate to the Plugins section and click on logging under the monitoring category.
  • Here you will define what needs to be logged according to the requirements
  • Ensure that the enable logging option is set to yes
  • Choose the default storage location for the log files.
  • Select additional details to be recorded, such as errors and warnings
  • Configure the log lifetime and expiry settings
  • Once you have made all the necessary changes, click on Save Changes
  • Logging is now enabled based on the set configurations. The logs will start recording the specified activities and stored in the chose storage path.
