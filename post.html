The 5 stages of software testing

Posted by Omar Baz on March 24, 2023

In this guide we will be following the standard Software Testing LifeCycle; which is a sequence of specific actions performed during the testing process to ensure that the software quality objectives are met. Following the correct sequence will ensure a successful deployment of the testing environment and proper results outputs for every specified test case.

The framework consits of 6 core principles:

  • 1. Requirements analysis
  • 2. Test planning
  • 3. Test Case Development
  • 4. Test Environment Setup
  • 5. Test Cycle Closure

In this guide we will be talking about test case development in detail, since it varies depending on what LMS you are targeting

1. Requirement analysis

The first stage of SLTC is requirement analysis. The main objective of this stage is to identify what needs to be tested and figure out the desired outcome. This can include architectural and design requirements, and system and integration requirements.
We will be conisdering an example of an LMS System where a student can register of different courses.
Examples of requirements can include:

  • Students will be able to enroll to undergraduate courses
  • A professor will log into the system by providing his username and password
  • A student will have either undergraduate or post graduate but not both
  • Register student and enrol courses webpages will load within 5 seconds
The requirements will vary from one system to another, based on the desired outcome and goals, as well as specific user experience parameters.

2. Test Planning

The second stage of SLTC is test planning. In this stage, the testing strategy should be defined. This includes noting down what types of tests will be covered and which will be ignored. Examples of test types include: unit testing, API testing, intergration testing and interface testing. The most important type to be defined in our case is unit testing, since we are focusing on testing plugins within an LMS environment.

3. Test Case Development

The third stage of SLTC is test case development. The main purpose of this stage is to create detailed test cases and prepare the data for testing. A comprehensive test case specifies the inputs, execution conditions, testing procedure, and expected results that all define a single test case to be executed in order to achieve the testing objectives. In this stage it is important to note how are we gonna structure the test and the desired objectives we are aiming to achieve.

4. Environment setup

The fourth stage of SLTC is the most important and relevant to us in this guide. The environment setup stage can be very time-consuming, but it's absolutely critical because it enables us to efficiently execute our test cases. In our case, we could either setup our enivronment locally or virtually on a server.

Both local and virtual testing environment share key elements that need to be applied to both:
  • Create test data (Dummy user Data)
  • Set up a database
  • Configure the environment

5. Test Closure

In the final step of the SLTC, we collect our test results. In this step you will be able to write down what you have learned about your plugin testing and point out what needs to be improved, taking lessons from the current test cycle. The idea is to remove bottlenecks for future test cycles, or to ensure that your plugin is ready for deployment based on your satisfaction with the results.
We will be also discussing creating surveys to collect user studies and interactions.
